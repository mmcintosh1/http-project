HTTP Requests and Responses

HTTP communication begins when a client, such as a web browser, sends a request to a web server. The server processes the request and sends back a response containing a status code, headers, and sometimes a message body.

Common HTTP Methods

  • GET: Retrieves information from a server.
  • POST: Sends new information to a server.
  • PUT: Updates or replaces an existing resource.
  • DELETE: Removes a resource from a server.
  • PATCH: Makes a partial update to a resource.

HTTP Status Codes

Status codes tell the browser whether a request succeeded, failed, or requires another action.

  • 100–199: Informational responses
  • 200–299: Successful responses
  • 300–399: Redirection responses
  • 400–499: Client errors
  • 500–599: Server errors

HTTP Headers

Headers provide additional details about a request or response. They can describe the type of content being sent, the browser being used, accepted languages, authorization information, and caching rules.

Cookies and Sessions

HTTP is stateless, which means each request is treated as a separate interaction. Cookies and sessions allow websites to remember login status, shopping-cart items, preferences, and other user information across multiple requests.

HTTP Caching

Caching allows browsers and servers to store copies of resources. When the same resource is requested again, a stored copy may be used, which can improve loading speed and reduce network traffic.
