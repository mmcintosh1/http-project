How HTTP Works

HTTP follows a client-server model. A web browser (the client) sends a request to a web server asking for a webpage, image, or other resource. The server processes the request and sends back a response containing the requested data along with a status code.

Every time you visit a website, many HTTP requests are exchanged in the background to load text, images, videos, and other content.

HTTP Request Process

  1. User enters a website address.
  2. Browser sends an HTTP request.
  3. Web server receives the request.
  4. Server processes the request.
  5. Server returns an HTTP response.
  6. Browser displays the webpage.

Common HTTP Status Codes

Status Code Meaning
200 Success
301 Moved Permanently
404 Page Not Found
500 Internal Server Error
