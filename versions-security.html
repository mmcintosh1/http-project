HTTP Versions

HTTP has gone through several major versions as the web has grown. Each version improved how browsers and servers communicate.

  • HTTP/0.9: The earliest version, used for simple HTML documents.
  • HTTP/1.0: Added headers, status codes, and more file types.
  • HTTP/1.1: Improved speed by allowing persistent connections.
  • HTTP/2: Improved performance by allowing multiple requests at once.
  • HTTP/3: Uses QUIC to improve speed and reliability.

HTTPS and Security

HTTPS is the secure version of HTTP. It uses encryption to protect information sent between a browser and a website. This is important for passwords, payment information, personal data, and login pages.

A website using HTTPS usually shows a lock icon in the browser address bar. This helps users know that the connection is more secure.
